# Portfolio Overview

This portfolio is organized as a professional research site, with each page aligned to core finance and machine learning themes.

## Portfolio Structure

The repository includes modular projects that can be executed independently while contributing to a cohesive analytics framework.

| Area | Project Folder | Purpose |
|---|---|---|
| Forecasting | `volatility-forecasting/` | Ensemble ML + GARCH volatility modeling and Monte Carlo simulations |
| Prediction | `prediction-model/` | Long/flat trading strategy for SPY using logistic regression and technical features |
| Sentiment | `sentiment-analysis/` | Hybrid sentiment scanner blending social, news, and technical signals |
| Correlation | `asset-correlation-analysis/` | Statistical analysis of Google Trends, tariffs, and stock price relationships |

## Architecture Principles

- **Modular design:** Each project folder contains isolated code, data, and pipelines.
- **Reproducibility:** Environment variables and requirements are centralized.
- **Data-driven:** Metrics, statistical tests, and visual analysis are core outputs.
- **Research-first:** Every model is framed with business context and measurable outcomes.

## Project Highlights

### Volatility Forecasting

A robust pipeline that combines:

- Polygon.io data ingestion
- Technical feature engineering
- ML regressors and ensemble methods
- GARCH volatility modeling
- Monte Carlo scenario simulation

### Trading Strategy Backtesting

A research-grade backtest that includes:

- Time series cross-validation
- Signal probability thresholds
- Sharpe ratio, max drawdown, and win rate
- Benchmark comparison vs buy-and-hold

### Quantitative Sentiment Research

A hybrid framework which blends:

- Reddit sentiment and comment volume
- Financial news scoring
- Technical indicator validation
- Configurable decision weights

### Correlation and Factor Modeling

A statistics-first study of:

- Search interest and equity returns
- Tariff discussions and retail performance
- Rolling correlations and significance testing
- Factor construction for alternative data

## Portfolio Narrative

This repository is designed to tell a story:

1. **Build trust** with clear data sources and pipeline descriptions.
2. **Demonstrate impact** through metrics and risk-based evaluation.
3. **Highlight innovation** using alternative data and hybrid model design.
4. **Keep it recruiter-friendly** with concise sectioning and business relevance.
